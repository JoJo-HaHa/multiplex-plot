4. Graph¶

Graphs, sometimes referred-to as networks, show the structure of data through nodes and edges. Each graph is made up of nodes and edges and can be directed or undirected.

Keep reading to learn more about graph visualizations.

Note

Check out the network graph Jupyter Notebook tutorial.

The Graph class can be used to draw directed or undirected graphs. Each graph is made up of nodes and edges and can be used to show the relations between nodes.

Note

The graph visualization uses the networkx package to generate the layout of the graph. Therefore it is a prerequisite to create these visualizations.

draw(G, positions=None, node_style=None, name_style=None, edge_style=None, label_style=None, *args, **kwargs)¶

Draw the given graph.

Any additional arguments and keyword arguments are passed on to the networkx.spring_layout() function.

Parameters

  • G (networkx.classes.graph.Graph) – The networkx graph to draw.

  • positions (dict) – The node’s initial positions. If they are given, the keys should be the node names and the values tuples representing their position. Nodes without a position are generated using network’x spring layout.

  • node_style (dict) – The general style for nodes. Keys correspond to the styling parameter and the values are the styling value. They are passed on as keyword arguments to the _draw_nodes() function. Individual nodes can override this style using the style attribute.

  • name_style (dict) – The general style for names. Keys correspond to the styling parameter and the values are the styling value. They are passed on as keyword arguments to the _draw_node_names() function. Individual names can override this style using the style attribute.

  • edge_style (dict) – The general style for edges. Keys correspond to the styling parameter and the values are the styling value. They are passed on as keyword arguments to the _draw_edges() function. Individual edges can override this style using the style attribute.

  • label_style (dict or None) – The style of the label.

Returns

A tuple containing the list of drawn nodes, the rendered node names, edges, and the rendered edge names.

Return type

tuple

_draw_nodes(nodes, positions, *args, **kwargs)¶

Draw the nodes onto the Drawable.

Any additional arguments and keyword arguments are passed on to the matplotlib.pyplot.scatter function.

The nodes should be given as dictionaries, whose keys are the node names. The corresponding values are their positions.

Parameters

  • nodes (networkx.classes.reportviews.NodeView) – The list of actual nodes to draw.

  • positions (dict) – The positions of the nodes as a dictionary. The keys are the node names, and the values are the corresponding positions.

Returns

A dictionary of rendered nodes. The keys are the node names and the values are matplotlib.collections.PathCollection, representing the rendered nodes.

Return type

dict

_draw_node_names(nodes, positions, s, *args, **kwargs)¶

Draw names for the nodes. Names are drawn if they have a name attribute. The name_style attribute, if given, is used to override the default name style. By default, names are aligned centrally and are positioned above the node.

Any additional keyword arguments are considered to be styling options.

Parameters

  • nodes (networkx.classes.reportviews.NodeView) – The list of nodes for which to draw names.

  • positions (dict) – The positions of the nodes as a dictionary. The keys are the node names, and the values are the corresponding positions.

  • s (float) – The default radius of the node. It may be overwritten with the node’s own radius.

Returns

A dictionary of rendered node names. The keys are the node names and the values are Annotation, representing the rendered annotations.

Return type

dict

_draw_edges(edges, nodes, positions, s, directed=False, *args, **kwargs)¶

Draw the edges connecting the given nodes. Depending on whether the graph is undirected or directed, the edges are drawn with arrows.

Any additional arguments and keyword arguments are passed on to the matplotlib.pyplot.plot function or as arrowprops.

Parameters

  • edges (list of tuple) – The list of edges to draw. The edges should be a list of tuples representing the source and target.

  • nodes (networkx.classes.reportviews.NodeView) – The list of graph nodes. These are not the rendered nodes, but the graph nodes.

  • positions (dict) – The positions of the nodes as a dictionary. The keys are the node names, and the values are the corresponding positions. They are used to connect the edges together.

  • s (float) – The default radius of the node. It may be overwritten with the node’s own radius.

  • directed (bool) – A boolean indicating whether the graph is directed or not.

Returns

A list of drawn edges. If the graph is undirected, lines are returned. Otherwise, annotations (with arrows) are returned.

Return type

list of matplotlib.lines.Line2D or list of matplotlib.text.Annotation

_draw_loop(node, position, s, directed, offset_angle=1.5707963267948966, *args, **kwargs)¶

Draw a loop, indicating an edge from a node to itself.

Any additional arguments and keyword arguments are passed on to the edge drawing functions.

Parameters

  • node (dict) – The node for which to draw a loop.

  • position (tuple) – The position of the node as a tuple.

  • s (float) – The default radius of the node. It may be overwritten with the node’s own radius.

  • directed – A boolean indicating whether the graph is directed or not.

  • offset_angle (float) – The loop’s offset angle in radians. By default, the value is \(\frac{\pi}{2}\), which places the loop on top of the node.

bool

Returns

A tuple containing the undirected edge and, if directed, the arrow.

_get_radius(node, s)¶

Get the radius of the given node in terms of the data axes. By default, the radius s is 100, but it can be overriden using the node’s style attribute.

Note

The square root of the radius is taken because the radius is originally squared.

Parameters

  • node (dict) – The node whose radius will be calculated.

  • s (float) – The default radius of the node. It may be overwritten with the node’s own radius.

Returns

The radius of the node in terms of the data axes. Two radii are provided: the x- and y-radii. This is because scatter points always look like circles, even when the display or data ratios are not equal. Whn the display or data ratios are not equal, the point is actually an ellipse so that it still looks like a circle.

Return type

tuple
