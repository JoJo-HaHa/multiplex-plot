4. Graph¶

Graphs, sometimes referred-to as networks, show the structure of data through nodes and edges. Each graph is made up of nodes and edges and can be directed or undirected.

Keep reading to learn more about graph visualizations.

Note

Check out the network graph Jupyter Notebook tutorial.

The Graph class can be used to draw directed or undirected graphs. Each graph is made up of nodes and edges and can be used to show the relations between nodes.

Note

The graph visualization uses the networkx package to generate the layout of the graph. Therefore it is a prerequisite to create these visualizations.

class graph.graph.Graph(*args, **kwargs)¶

The Graph class draws nodes and edges.

__init__(*args, **kwargs)¶

Initialize the graph.

draw(G, positions=None, node_style=None, name_style=None, edge_style=None, label_style=None, *args, **kwargs)¶

Draw the given graph.

Any additional arguments and keyword arguments are passed on to the networkx.spring_layout() function.

Parameters

  • G (networkx.classes.graph.Graph) – The networkx graph to draw.

  • positions (dict) – The node’s initial positions. If they are given, the keys should be the node names and the values tuples representing their position. Nodes without a position are generated using network’x spring layout.

  • node_style (dict) – The general style for nodes. Keys correspond to the styling parameter and the values are the styling value. They are passed on as keyword arguments to the _draw_nodes() function. Individual nodes can override this style using the style attribute.

  • name_style (dict) – The general style for names. Keys correspond to the styling parameter and the values are the styling value. They are passed on as keyword arguments to the _draw_node_names() function. Individual names can override this style using the style attribute.

  • edge_style (dict) – The general style for edges. Keys correspond to the styling parameter and the values are the styling value. They are passed on as keyword arguments to the _draw_edges() function. Individual edges can override this style using the style attribute.

  • label_style (dict or None) – The style of the label.

Returns

A tuple containing the list of drawn nodes, the rendered node names, edges, and the rendered edge names.

Return type

tuple
