4. Graph¶

Graphs, sometimes referred-to as networks, show the structure of data through nodes and edges. Each graph is made up of nodes and edges and can be directed or undirected.

Keep reading to learn more about graph visualizations.

Note

Check out the network graph tutorial Jupyter Notebook.

The Graph class can be used to draw directed or undirected graphs. Each graph is made up of nodes and edges and can be used to show the relations between nodes.

Note

The graph visualization uses the networkx package to generate the layout of the graph. Therefore it is a prerequisite to create these visualizations.

class graph.graph.Graph(*args, **kwargs)¶

The Graph class draws nodes and edges.

__init__(*args, **kwargs)¶

Initialize the graph.

draw(G, node_style=None, name_style=None, edge_style=None, *args, **kwargs)¶

Draw the given graph.

Any additional arguments and keyword arguments are passed on to the networkx.spring_layout() function.

Parameters

  • G (networkx.classes.graph.Graph) – The networkx graph to draw.

  • s (float) – The size of the nodes.

  • node_style (dict) – The general style for nodes. Keys correspond to the styling parameter and the values are the styling value. They are passed on as keyword arguments to the _draw_nodes() function. Individual nodes can override this style using the style attribute.

  • name_style (dict) – The general style for names. Keys correspond to the styling parameter and the values are the styling value. They are passed on as keyword arguments to the _draw_node_names() function. Individual names can override this style using the style attribute.

  • edge_style (dict) – The general style for edges. Keys correspond to the styling parameter and the values are the styling value. They are passed on as keyword arguments to the _draw_edges() function. Individual edges can override this style using the style attribute.

Returns

A tuple containing the list of drawn nodes, their names, and edges.

Return type

tuple

_draw_nodes(nodes, positions, *args, **kwargs)¶

Draw the nodes onto the Drawable.

Any additional arguments and keyword arguments are passed on to the matplotlib.pyplot.scatter() function.

The nodes should be given as dictionaries, whose keys are the node names. The corresponding values are their positions.

Parameters

  • nodes (networkx.classes.reportviews.NodeView) – The list of actual nodes to draw.

  • positions (dict) – The positions of the nodes as a dictionary. The keys are the node names, and the values are the corresponding positions.

Returns

The list of rendered nodes.

Return type

matplotlib.collections.PathCollection

_draw_node_names(nodes, positions, *args, **kwargs)¶

Draw labels for the nodes. Labels are drawn if they have a name attribute. The name_style attribute, if given, is used to override the default label style. By default, nodes are aligned centrally and are positioned above the node.

Any additional keyword arguments are considered to be styling options.

Parameters

  • nodes (networkx.classes.reportviews.NodeView) – The list of nodes for which to draw labels.

  • positions (dict) – The positions of the nodes as a dictionary. The keys are the node names, and the values are the corresponding positions.

Returns

The list of drawn labels.

Return type

list of Annotation

_draw_edges(edges, nodes, *args, **kwargs)¶

Draw the edges connecting the given nodes.

Any additional arguments and keyword arguments are passed on to the matplotlib.pyplot.plot() function.

Parameters

  • edges (list of tuple) – The list of edges to draw. The edges should be a list of tuples representing the source and target.

  • nodes (dict) – The nodes in terms of their positions as a dictionary. The keys are the node names, and the values are the corresponding positions.

Returns

A list of drawn edges.

Return type

list of matplotlib.lines.Line2D
